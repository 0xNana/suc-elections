# Audit and Verification

This document describes the intended verification path before public results are shown.

## Result Sequence

1. Polls close
2. Electoral Commission generates the official count
3. Aspirant reps review the count
4. Aspirant reps submit verification messages
5. Electoral Commission releases the results to the public

## What Aspirant Reps Should Check

After EC counts results, reps should:

- confirm the candidate totals
- review the audit trail
- watch for repeated activation resets
- watch for unexpected role changes
- watch for late schedule changes

If satisfied, the rep submits a verification message.

## What EC Should Check Before Release

Before releasing results, EC should confirm:

- polls are closed
- the official count exists
- at least one rep verification exists
- there are no unresolved anomalies in the audit trail

## Evidence Recorded

The system records:

- code issuance
- account activation
- logins
- poll open
- poll close
- result counting
- rep verification
- public result release
